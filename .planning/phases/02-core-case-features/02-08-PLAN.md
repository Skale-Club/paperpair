**Step 1: Update src/app/documentation-filling/page.tsx**

Read the file first. The page already calls `getCurrentUserAndProfile()` — wait, it doesn't based on the current code. It only fetches templates. We need to fetch chat history in this server component.

The page needs to:
1. Import `getCurrentUserAndProfile` from `@/lib/current-user-profile`
2. Fetch chat history from `/api/chat/history`
3. Pass history as `initialMessages` to `ChatUI`

Since this is a Server Component (not a Client Component), use `fetch` with the internal URL pattern. However, in Next.js App Router server components, calling internal API routes is an anti-pattern. Instead, call the database directly using the same pattern as the history route:

```typescript
// Add this import
import { getCurrentUserAndProfile } from "@/lib/current-user-profile";
import { prisma } from "@/lib/prisma";
```

Add inside the page function, BEFORE the return:
```typescript
// Fetch chat history — per D-13, CHAT-04
type InitialMessage = { id: string; role: "user" | "assistant"; content: string };
let initialMessages: InitialMessage[] = [];

const userContext = await getCurrentUserAndProfile();
if (userContext) {
  try {
    const session = await prisma.chatSession.findUnique({
      where: { userProfileId: userContext.userProfile.id },
      include: { messages: { orderBy: { createdAt: "asc" } } },
    });
    if (session) {
      initialMessages = session.messages
        .filter((m) => m.role === "user" || m.role === "assistant")
        .map((m) => ({
          id: m.id,
          role: m.role as "user" | "assistant",
          content: m.content,
        }));
    }
  } catch {
    initialMessages = [];
  }
}
```

Update the ChatUI call to pass `initialMessages`:
