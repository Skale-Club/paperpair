```prisma
model ChatSession {
  id            String        @id @default(cuid())
  userProfileId String        @unique
  createdAt     DateTime      @default(now())
  updatedAt     DateTime      @updatedAt
  messages      ChatMessage[]

  userProfile   UserProfile   @relation(fields: [userProfileId], references: [id], onDelete: Cascade)
}

model ChatMessage {
  id        String      @id @default(cuid())
  sessionId String
  role      String
  content   String
  createdAt DateTime    @default(now())

  session   ChatSession @relation(fields: [sessionId], references: [id], onDelete: Cascade)
}
```

Notes:
- `userProfileId @unique` enforces the D-01 constraint (one session per user)
- `role` is String (not enum) — values will be "user" | "assistant" per D-03
- No model ID / token count fields per D-03
- `onDelete: Cascade` on both relations ensures cleanup when UserProfile is deleted

After editing the schema file, run the migration:
```

```

This generates the migration SQL in prisma/migrations/ and regenerates the Prisma client.
