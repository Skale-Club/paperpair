## Color

| Role | Value | CSS Variable | Usage |
|------|-------|-------------|-------|
| Dominant (60%) | #f7f9f3 | `var(--color-bg)` | Page background, shell background |
| Secondary (30%) | #ffffff | `var(--color-surface)` | Cards, panel backgrounds, form inputs |
| Accent (10%) | #6c7b4e | `var(--color-trust)` | Reserved list below |
| Destructive | #dc2626 | `var(--color-destructive)` | Delete document button only |

**Accent reserved for:**
1. Primary CTA button fill (screener "Next" button, upload trigger, "Extract to profile" button)
2. Active sidebar navigation item indicator
3. Checkbox fill when checked
4. Progress bar fill
5. UPL disclaimer left-border stripe (existing pattern — preserve)

**Accent is NOT used for:** cards, section headings, body text, secondary actions, icon colors, or hover states.

**Warning color (not accent):** EWI inline warning and form edition warning banner use `var(--color-warning-bg)` / `var(--color-warning-border)` / `var(--color-warning-text)` — amber tones, not olive.

**Slate tones** (from Tailwind defaults) continue to be used for tertiary text (`text-slate-500`), borders (`border-slate-200`), and background washes (`bg-slate-50`). This is an existing pattern — do not replace with custom tokens.

## Interaction Contracts

### Screener Completion (CASE-01, CASE-02)

- Screener step buttons use `var(--color-trust)` fill with white text (existing pattern).
- On step 8 completion: save entry type to `immigration-info` CaseStep → screener dismisses → main dashboard renders. No intermediate success screen.
- EWI warning: shown inline below the screener card before it dismisses (not a modal, not a toast). Uses `var(--color-warning-bg)` amber banner. Persists on dashboard until dismissed or until user navigates away.
- Loading state during save: "Next" button shows disabled state with `opacity-40` and cursor-not-allowed. No spinner — text changes to "Saving…".

### Forms Panel (FORM-01 through FORM-04)

- Pack detail panel opens as a side drawer or inline expansion (existing UX pattern — do not change the open/close mechanic).
- Inside panel, instructions section layout (top to bottom): form purpose (1-2 sentences in body text), "Who fills this" label row, "What to expect" paragraph, edition warning banner (amber, conditional), "Download from USCIS.gov" external link button.
- Edition warning banner: `border border-amber-200 bg-amber-50 rounded-lg px-4 py-3 text-sm text-amber-800` with a warning icon SVG. Text: "This edition may be outdated. Always download the current form directly from USCIS.gov before filing."
- USCIS download link: `<a href={pack.uscisUrl} target="_blank" rel="noopener noreferrer">` styled as a secondary button (border, not filled).

### Documents Page (DOC-01 through DOC-05)

- Layout: full-width list. No gallery/grid. Each row is one document.
- Upload trigger: dashed border drop zone (`border-2 border-dashed rounded-xl`) with a centered "Upload document" button (accent fill, 44px height).
- Doc-type tag selector: appears immediately after file selection, before upload commits. Native `<select>` styled with `rounded-md border border-slate-300 bg-white px-3 py-2 text-sm`. Required field — upload button disabled until type is selected.
- Document row: `flex items-center justify-between rounded-lg border border-slate-200 bg-white px-4 py-3`. Contents: file icon (SVG) + filename (truncated with `truncate max-w-[200px]`) + type badge + date + download button + delete button.
- Type badge: `rounded-full bg-[var(--color-trust-muted)] px-2 py-0.5 text-xs font-medium text-[var(--color-muted)]`.
- Download button: icon-only with accessible label, `text-slate-500 hover:text-slate-900`.
- Delete button: icon-only with accessible label, `text-slate-400 hover:text-[var(--color-destructive)]`.
- "Extract to profile" button: appears per-row only when doc type is `passport`, `marriage-certificate`, or `birth-certificate` (types with extraction support). Label: "Extract to profile". Accent fill, `text-xs`, 32px height.
- Uploading state: row shows skeleton placeholder with `animate-pulse bg-slate-100`.
- Delete confirmation: inline — clicking delete replaces the row's action buttons with "Confirm delete?" + "Yes, delete" (destructive) + "Cancel" (neutral) for 5 seconds, then auto-reverts. No modal.

### Chat (CHAT-04, CHAT-06)

- Chat history loads on mount — no user-visible loading spinner if messages arrive within 500ms. If > 500ms, show skeleton rows (3 alternating left/right placeholder bubbles, `animate-pulse`).
- AP travel warning: rendered as a standard AI assistant message bubble (existing bubble styles). No special banner treatment — the AI message itself contains the warning copy.
- Chat input remains unchanged (existing `chat-input.tsx` pattern).

## Accessibility Requirements

- All icon-only buttons (download, delete) must have `aria-label` set to the action name.
- The EWI warning uses `role="alert"` so screen readers announce it on appearance.
- The edition warning banner uses `role="note"` (same pattern as `UplDisclaimer`).
- Upload drop zone keyboard-activatable via `<label>` wrapping `<input type="file">`.
- Doc-type `<select>` associated with visible `<label>` via `htmlFor` / `id`.
- Delete confirmation timeout (5s auto-revert) must be announced via `aria-live="polite"` region.
- Focus management: after screener dismisses, focus moves to first dashboard heading.
- Color alone is never the only indicator of state — type badges and warning banners include text labels.
