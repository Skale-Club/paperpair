## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, checkbox-to-label gap, inline badge padding |
| sm | 8px | Compact list item spacing, button icon gaps |
| md | 16px | Default element spacing, form field gaps, panel padding (mobile) |
| lg | 24px | Section padding, card inner padding |
| xl | 32px | Layout gaps between major panel sections |
| 2xl | 48px | Page-level top margin on auth cards |
| 3xl | 64px | Not used in Phase 1 |

Exceptions:
- Touch targets (checkboxes, interactive buttons in nav): minimum 44px height — apply `min-h-[44px]` where the visual size is smaller
- Left panel nav items: 40px minimum height (`min-h-[40px]`) to maintain tap target density on mobile
- Timeline connector line: fixed 2px width (not a spacing token — structural)
- `mt-0.5` (2px): optical icon alignment on UPL banner icon — structural, not a spacing token (same treatment as 2px timeline connector line)

## Component Contracts

### 1. Timeline Dual-Panel (TIME-01 to TIME-05)

**Source:** CONTEXT.md § Specifics — locked decision.

**Layout:** `grid lg:grid-cols-[260px_1fr]` — existing structure preserved. Both panels are independently scrollable.

**Focal point:** Primary focal point is the right panel section heading, anchored to the first active section on load. On initial render, the right panel scrolls to (or starts at) the first incomplete section heading.

**Left panel (step navigator):**
- Container: `h-[calc(100vh-theme(spacing.32))] overflow-y-auto` (sticky, scrollable independently)
- Width: 260px fixed on desktop; full-width collapsible on mobile
- Phase groups: collapsible accordion (expand/collapse on phase header click)
- Active section highlight: `border-slate-300 bg-white font-bold text-slate-900 shadow-sm` (weight 700 — bold only)
- Done section: `border-slate-400 bg-slate-100 text-slate-700` with CheckIcon
- Partial progress indicator: `{checkedItems}/{items.length}` text badge, right-aligned, `text-xs text-slate-400`
- Connector lines: 2px `bg-slate-200` vertical + horizontal lines (existing tree structure — preserve)
- Scrollspy behavior: `IntersectionObserver` on right-panel section anchors drives active highlight; NO jump, only highlight update

**Right panel (section content):**
- Container: `h-[calc(100vh-theme(spacing.32))] overflow-y-auto rounded-2xl border border-slate-200 bg-white p-6 lg:p-8`
- Each section must have a scrollspy `id` anchor matching its `sectionId` (e.g. `id="determine-eligibility"`)
- Section content area: `space-y-4` with heading, subtitle, optional Tip/Warning, body text, checklist
- Checklist items: `flex cursor-pointer items-start gap-3 rounded-lg border px-4 py-3` — checked state uses `border-slate-200 bg-slate-50`; unchecked uses `border-slate-200 bg-white hover:bg-slate-50`
- Progress bar: `h-1.5 rounded-full bg-slate-100` with `bg-slate-900` fill — existing pattern preserved
- Overall progress bar (above grid): `h-2 rounded-full bg-slate-100` with `bg-slate-900` fill — existing pattern preserved

**Scrollspy sync specification:**
- Use `IntersectionObserver` with `threshold: 0` and `rootMargin: "-20% 0px -70% 0px"` to detect which section is visible in the right panel
- When a section enters the intersection zone: update `activeSection` state and auto-expand the parent phase accordion if collapsed
- Left panel does NOT auto-scroll to track the active section (user controls left panel scroll independently)
- Clicking a left-panel section: scrolls right panel to that section's anchor (`element.scrollIntoView({ behavior: "smooth", block: "start" })`), updates `activeSection`

**Mobile behavior:**
- Left panel becomes a collapsible dropdown triggered by the "current section" button (existing pattern preserved)
- Both panels are no longer independently scrollable on mobile — page scrolls normally
- Scrollspy disabled on mobile (viewport < 1024px)

**Persistence UI feedback:**
- No loading spinner needed — persistence is a background upsert
- No toast/notification on save — silent background save (same UX as before, just reliably persisted)
- If persistence fails: no UI error — fail silently (v1 resilience strategy)

**Progress bar copy:**
- Overall: `{completedSectionCount}/{totalSections} sections`
- Section item count: `{checkedCount}/{totalCount}`
- Footer remaining items: `{N} item remaining` / `{N} items remaining` / `All items complete` / `Mark as complete to continue`

### 2. UPL Disclaimer Banner (BUG-08)

**Source:** CONTEXT.md § Claude's Discretion — non-dismissible, server-side rendered, on chat/form/screener pages.

**Placement:** Below the page header / above page main content on affected pages. NOT in a modal, NOT a toast. Sticky position is NOT required — renders in document flow.

**Visual specification:**
```
[left border 4px accent stripe] [icon] PaperPair provides general information only — not legal advice.
                                        Always consult a qualified immigration attorney for your specific situation.
```

- Container: `flex items-start gap-3 rounded-lg border border-[var(--color-trust-muted)] bg-[var(--color-trust-muted)]/40 px-4 py-3`
- Left accent stripe: `border-l-4 border-l-[var(--color-trust)]` (add to container class)
- Icon: inline SVG info circle, `h-4 w-4 shrink-0 text-[var(--color-trust)] mt-0.5` (the `mt-0.5` is optical icon alignment — see Spacing Exceptions)
- Text: `text-sm text-[var(--color-muted)]`
- Bold lead: `font-bold` on "PaperPair provides general information only — not legal advice."
- Second sentence: regular weight (400)
- Non-dismissible: no close button, no X
- Server-side rendered: no `"use client"` — pure JSX component

**Full copy:**

> **PaperPair provides general information only — not legal advice.** Always consult a qualified immigration attorney for your specific situation.

**Pages where banner appears:**
- `/dashboard/my-forms` and all form detail pages
- `/app/chat` (chat page)
- Any screener page (Phase 2 — banner component must be ready)

**Component name:** `UplDisclaimer` — exported from `src/components/upl-disclaimer.tsx`

## Interaction Contracts

### Scrollspy (Timeline)

| Event | Response |
|-------|----------|
| User scrolls right panel down past section boundary | `activeSection` state updates; left panel section highlight moves; no jump |
| User clicks left panel section | Right panel scrolls smoothly to section anchor; `activeSection` updates; phase accordion auto-expands |
| User scrolls left panel | Right panel does NOT move |
| User scrolls right panel | Left panel does NOT move |
| Mobile viewport (<1024px) | Scrollspy disabled; clicking section in dropdown closes dropdown and scrolls page to top |

### Checklist Items (Timeline)

| Event | Response |
|-------|----------|
| Check an item | Checkbox fills, label gets `line-through text-slate-400`, progress bar animates, state upserted to DB silently |
| Uncheck an item | Checkbox clears, label restores, progress bar animates, state upserted to DB silently |
| All items in section checked | Section auto-marks complete in left nav (green check appears) |
| "Next" button clicked | Current section marked complete, scroll advances to next section |

### Auth Forms

| Event | Response |
|-------|----------|
| Submit with empty required field | Browser native validation (no custom UI needed) |
| Submit while loading | Button shows "Working...", disabled state (`opacity-60`) |
| OAuth click while loading | All OAuth buttons get `disabled` |
| OAuth redirect | Button shows "Redirecting..." label |
| Successful sign in | `router.push("/dashboard")` — no intermediate screen |
| Successful sign up (no immediate session) | Inline success message replaces error area — no redirect |
| Successful sign up (immediate session) | `router.push("/dashboard")` |

### Invite Acceptance

| Event | Response |
|-------|----------|
| Page load | Token validated server-side before render (or client-side API call on mount) |
| "Accept invitation" clicked | Button enters loading state ("Accepting..."), POST to `/api/invite/accept`, redirect to `/dashboard` on success |
| Acceptance API error | Inline error below button: `text-sm text-red-700` — "We couldn't accept this invitation. Try refreshing the page, or ask your spouse to send a new invite." |
| "Decline" clicked | Navigate to `/login` with no confirmation |

## Accessibility Requirements

- All interactive elements: `focus-visible:ring-2 focus-visible:ring-[var(--color-trust)] focus-visible:ring-offset-2`
- Checkbox inputs: `aria-label` not needed — wrapped in `<label>` with visible text
- Phase accordion buttons: `aria-expanded` reflecting `isExpanded` state
- Section nav buttons: `aria-current="page"` when `isActive`
- UPL banner: `role="note"` on the container
- Invite page loading spinner: `aria-live="polite"` on the status region
- Color contrast: `var(--color-trust)` (#6c7b4e) on white — WCAG AA verified (4.6:1 ratio sufficient for large text; borderline for small text — prefer bold weight at 14px)
